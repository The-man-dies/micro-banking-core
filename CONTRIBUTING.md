# Contributing

Thanks for contributing to Micro Banking Core.

## Workflow

1. Fork the repository.
2. Create a dedicated branch from `main`.
3. Keep commits focused and atomic.
4. Open a pull request with a clear summary, rationale, and test evidence.

## Branch Naming

Examples:

- `feat/<short-description>`
- `fix/<short-description>`
- `chore/<short-description>`
- `docs/<short-description>`

## Commit Messages

Use concise conventional-style messages, for example:

- `feat(desktop): add sidecar readiness check`
- `fix(server): include prisma migrations in runtime`
- `docs(readme): clarify release process`

## Pull Request Checklist

- CI is green.
- Scope is limited to the intended change.
- No unrelated refactor included.
- Documentation updated when behavior or workflow changed.
- Screenshots/logs attached when relevant.

## Development Notes

- Current status: dev mode is unstable/broken.
- Use production bundle validation for acceptance testing.

## Code Style

- Follow existing project conventions.
- Keep changes minimal and explicit.
- Avoid hidden behavior changes.

## Reporting Bugs

For bug reports, include:

- Platform/OS
- App version
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs/screenshots
